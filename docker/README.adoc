All docker commands are run from the `docker` directory.

== Copy the sample docker env file

[,shell]
----
just setup-dev
# Edit the .env to your preferences
$EDITOR .env
----

Edit the .env file for your choice of p2poolv2 ports etc.

== Start your local p2pool node

[,shell]
----
cd docker
docker compose up -d
----

== Shutdown your local p2pool node

[,shell]
----
docker compose down
----

== Running multiple P2Poolv2 nodes

First, start the shared infrastructure (bitcoind + default node):

[,shell]
----
docker compose up -d
----

Then start additional nodes, each with its own .env and project name:

[,shell]
----
docker compose -f docker-compose.node.yml -p node2 --env-file .env.node2 up -d
----

[,shell]
----
docker compose -f docker-compose.node.yml -p node3 --env-file .env.node3 up -d
----

To stop an additional node:

[,shell]
----
docker compose -f docker-compose.node.yml -p node2 down
----

== Building with Podman

If you are using Podman instead of Docker, an extra step is required to build
the P2Poolv2 container image. This is due to a limitation with Dockerfile
syntax support in Podman. See the related issue: [https://github.com/containers/podman/issues/26458[containers/podman#26458].

First, start a BuildKit daemon:

[,shell]
----
podman run -d --name buildkitd --privileged docker.io/moby/buildkit:latest
----

Then build the image from the project root:

[,shell]
----
buildctl \
  --addr=podman-container://buildkitd \
  build \
  --frontend=gateway.v0 \
  --opt source=docker/dockerfile:1.20 \
  --local context=. \
  --local dockerfile=./docker/p2poolv2/ \
  --output "type=docker,name=p2poolv2:latest" \
  | podman load
----
